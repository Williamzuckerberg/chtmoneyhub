Asterisk，freeswitch，pjsip打造扩展性极强的VOIP系统

Asterisk，freeswitch，opal，pjsip等优秀的开源项目，对于开发先进的VOIP电话系统提供了便捷和廉价的途径。
sun基于VOIP项目实践的经验，从技术上对VOIP电话系统的开发思路进行一个分析。

以下是设计思路:
利用Asterisk（freeswitch）作为VOIP电话的服务器，pjsip作为客户端，UI端通过socket(java,C,vc,etc)和pjsip进行通讯。架构图如下

 服务端                                                          客户端
Asterisk（freeswitch）<===============>pjsip<=========socket(java/C)===================>UI
  |                                                                             |                             |                                                          |
 mysql                                                   pjsua(1.x or later)             WSASocket(Win)                         Java(SWING)     
电话配置                                              更改pjsua_app.c等        Socket(Linux)                                JAVA<->Flex
电话计费                                                                                                                                           Softphone
                                                                                                                                                       呼叫中心 Call center(frontdesk)


1,Asterisk（freeswitch
1.1) 需要把账号的配置信息存入数据库，可以通过Web 界面进行方便的管理和配置。或简单配置一下配置文件（不用数据库）。
1.2) 产生通话记录历史（cdr）,为计费和通话统计服务
1.3) 适当的情况下，可以扩展到多Asterisk服务器来支持扩充

2，pjsip
2.1) pjsip是客户端和服务端通讯的核心，实际上，这个设计。pjsip这一端可以和UI整合为一个软电话，也可以分离，作为比较复杂应用的核心，对接好几个UI如IM，WEB应用等。
2.2) 需要适当修改pjsip的源代码，支持Socket通讯（Server Socket/ Client Socket）. pjsip担当Server Socket的角色，监听来自UI端的请求；同时把来自UI端的请求发往电话服务器(Asterisk).

3,Socket
Socket 可以用使Java，C等来实现。它作为UI和pjsip之间的桥接。

4，UI
4.1) UI客户端是客户真正看到的东西。比如Skype界面就是UI的一种展现。
我们可以根据实际情况，用不同的程序语言来实现UI端的开发。
